# 🏗️ Atividade - Pesquisa de Arquiteturas de Software

> **Curso:** DevOps - Aponti
> **Modalidade:** Atividade prática

---

## 📖 Sobre

Esta atividade teve como objetivo pesquisar sistemas reais utilizados por empresas e identificar qual arquitetura de software foi adotada por cada um — **Monolítica**, **SOA** ou **Microsserviços** — compreendendo os motivos da escolha, suas vantagens, limitações e contexto de utilização.

Para cada arquitetura foi pesquisado um sistema real de mercado, com base em artigos técnicos e acadêmicos, e ao final foi elaborada uma análise comparativa entre as três abordagens.

---

## 🎯 Objetivos

- Compreender na prática as diferenças entre arquitetura Monolítica, SOA e Microsserviços;
- Relacionar cada arquitetura a um sistema real de mercado;
- Identificar vantagens, desafios e motivações por trás da escolha arquitetural de cada empresa;
- Desenvolver pensamento crítico para comparar arquiteturas e justificar qual se encaixa melhor em diferentes cenários.

---

## 📋 Sistemas pesquisados

Foram pesquisados três sistemas reais, um para cada arquitetura:

- 🧱 **Shopify** — Arquitetura Monolítica (monólito modular);
- 🏦 **Banco do Brasil** — Arquitetura SOA (integração de sistemas legados via barramento de eventos/serviços);
- 🎬 **Netflix** — Arquitetura de Microsserviços.

Para cada sistema foram respondidas as seguintes questões:

- Qual é a finalidade deste sistema?
- Qual arquitetura ele utiliza?
- Como essa arquitetura funciona neste sistema?
- Por que essa arquitetura foi escolhida?
- Quais são as principais vantagens dessa arquitetura?
- Quais desafios ou limitações ela apresenta?
- Esse sistema poderia utilizar outra arquitetura?
- Curiosidade encontrada durante a pesquisa.

---

## 🎯 Ficha de Conclusão

Além da pesquisa individual de cada sistema, foi elaborada uma ficha de conclusão comparando as três arquiteturas, considerando aspectos como:

- Simplicidade de compreensão e implementação;
- Adequação a pequenas ou grandes empresas;
- Facilidade de escalabilidade;
- Investimento necessário em infraestrutura;
- Alinhamento com a cultura DevOps;
- Escolha ideal para uma startup hoje.

---

## 🛠️ Conceitos aplicados

- Arquitetura Monolítica;
- Arquitetura Orientada a Serviços (SOA);
- Arquitetura de Microsserviços;
- Escalabilidade;
- Barramento de serviços (ESB);
- Cultura DevOps;
- Deploy contínuo.

## 💡 Aprendizados

Esta atividade permitiu compreender como diferentes empresas escolhem sua arquitetura de software de acordo com o estágio, a escala e as necessidades do negócio — desde o monólito modular do Shopify, passando pela integração de sistemas legados via SOA no Banco do Brasil, até a arquitetura de microsserviços da Netflix, evidenciando que não existe uma arquitetura "melhor" de forma absoluta, e sim mais adequada a cada contexto.
