# 🛠️ Atividade 02 - Estudo de Caso DevOps

> **Curso:** DevOps - Aponti  
> **Modalidade:** Atividade prática

---

## 📖 Sobre

Esta atividade teve como objetivo analisar situações comuns encontradas em empresas de desenvolvimento de software e propor soluções utilizando os princípios da cultura DevOps.

Assumindo o papel de um consultor DevOps, foi necessário identificar problemas, compreender suas causas e sugerir práticas, ferramentas e conceitos capazes de tornar os processos mais eficientes.

---

## 🎯 Objetivos

- Identificar gargalos em processos de desenvolvimento;
- Relacionar problemas reais aos conceitos de DevOps;
- Aplicar práticas utilizadas pelo mercado;
- Desenvolver pensamento crítico na resolução de problemas.

---

## 📋 Problemas analisados

Durante a atividade foram analisados cinco cenários:

- 🚀 Deploy Manual;
- 🤝 Falta de Comunicação entre equipes;
- 🔀 Integração de Código;
- ✅ Testes Manuais;
- 🖥️ Diferenças entre ambientes de desenvolvimento e produção.

Para cada situação foram respondidas as seguintes questões:

- Qual é o principal problema?
- Quais são as possíveis causas?
- Como a cultura DevOps pode ajudar?
- Qual prática, ferramenta ou conceito pode ser aplicado?
- Quais benefícios essa solução traria para a empresa?

---

## 🎯 Desafio Final

Além da análise dos problemas, foi elaborado um fluxo comparando o processo tradicional de desenvolvimento com um fluxo baseado nos princípios DevOps. O novo fluxo considera conceitos como:

- Colaboração entre equipes;
- Integração Contínua (CI);
- Entrega Contínua (CD);
- Testes automatizados;
- Monitoramento;
- Feedback contínuo.

---

## 🛠️ Conceitos aplicados

- Cultura DevOps;
- CI/CD;
- Automação;
- Git;
- GitHub;
- Trabalho colaborativo;
- Monitoramento;
- Infraestrutura como Código (IaC).
